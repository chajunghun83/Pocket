# Pocket - 개인 재무 관리 프로그램 PRD

## 1. 프로젝트 개요

### 1.1 목적
개인의 수입/지출, 부채, 주식을 통합적으로 관리할 수 있는 가계부 애플리케이션

### 1.2 목표
- 월별 수입/지출 현황을 한눈에 파악
- 부채 관리를 통한 재무 상태 추적
- 보유 주식의 실시간 손익 확인
- 장기적인 재무 상태 분석을 위한 통계 제공

---

## 2. 핵심 기능

### 2.1 가계부 관리

#### 2.1.1 고정 비용 관리
**기능 설명**
- 매월 정기적으로 발생하는 고정 지출 항목을 사전에 등록하고 관리

**주요 기능**
- 고정 지출 항목 등록 (항목명, 금액, 출금일)
- 항목별 출금 완료 체크 기능
- 미출금 항목 확인 기능
- 고정 지출 항목 수정/삭제

**예시**
- 월세, 관리비, 통신비, 보험료, 구독 서비스 등

#### 2.1.2 변동 비용 관리
**기능 설명**
- 카드 사용 등 변동적으로 발생하는 지출 관리

**주요 기능**
- 예정 지출 항목 등록 (항목명, 금액, 예정일)
- 출금 완료 체크 기능
- 변동 지출 항목 추가/수정/삭제

**예시**
- 카드 결제액, 일회성 지출 등

#### 2.1.3 수입 관리
**기능 설명**
- 월급, 부수입 등 수입 내역 관리

**주요 기능**
- 수입 항목 등록 (항목명, 금액, 입금일)
- 입금 완료 체크 기능
- 수입 항목 추가/수정/삭제

#### 2.1.4 월별 수지 계산
**기능 설명**
- 해당 월의 총 수입과 총 지출을 계산하여 수지 현황 표시

**표시 항목**
- 총 수입
- 총 지출 (고정 비용 + 변동 비용)
- 수지 (수입 - 지출)
- 잔액 상태 표시 (흑자/적자)

#### 2.1.5 통계 기능
**기능 설명**
- 월별 수입/지출 추이를 시각적으로 표시

**주요 기능**
- 월별 수입/지출 그래프
- 월별 수지 추이 차트
- 지출 카테고리별 비중 (선택)
- 기간별 비교 (3개월, 6개월, 1년)

### 2.2 부채 관리

#### 2.2.1 마이너스 통장 관리
**기능 설명**
- 마이너스 통장의 대출/상환 내역을 기록하고 잔액을 추적

**주요 기능**
- 대출 내역 등록
  - 날짜
  - 금액
  - 내용/메모
- 상환 내역 등록
  - 날짜
  - 금액
  - 내용/메모
- 현재 잔액 자동 계산 및 표시
- 대출/상환 내역 수정/삭제

**표시 항목**
- 총 대출액
- 총 상환액
- 현재 부채 잔액
- 거래 내역 리스트 (시간순)

#### 2.2.2 부채 통계
**기능 설명**
- 부채 추이를 시각적으로 표시

**주요 기능**
- 월별 부채 잔액 추이 그래프
- 월별 대출/상환 금액 비교
- 기간별 부채 증감 분석

### 2.3 주식 관리

#### 2.3.1 보유 주식 관리
**기능 설명**
- 보유 중인 주식 정보를 입력하고 현재 손익을 확인
- **국장(한국)과 미장(미국) 동시 지원**

**주요 기능**
- 주식 정보 등록
  - 시장 구분 (🇰🇷 국장 / 🇺🇸 미장)
  - **국장**: 기업명 (종목코드) - 예: 삼성전자 (005930)
  - **미장**: 기업명 (티커) - 예: Apple (AAPL)
  - 보유 수량
  - 평균 매수가(평단가)
  - 통화 (KRW/USD)
- 주식 정보 수정
  - 추가 매수 시 평단가 자동 계산
  - 일부 매도 시 수량/평단가 조정
- 주식 정보 삭제

#### 2.3.2 손익 계산
**기능 설명**
- 현재가와 평단가를 비교하여 평가 손익 표시
- 국장과 미장 주식을 통합하여 포트폴리오 관리

**표시 항목**
- 종목별 정보
  - 시장 구분 (🇰🇷/🇺🇸)
  - **종목 표시**
    - 국장: 기업명 (종목코드) - 예: 삼성전자 (005930)
    - 미장: 기업명 (티커) - 예: Apple (AAPL)
  - 보유 수량
  - 평균 매수가 (원화 또는 달러)
  - 현재가 (자동 조회, 원화 또는 달러)
  - **개별 종목은 원래 통화로 표시**
    - 국장: 원화 (₩)
    - 미장: 달러 ($)
  - 평가 금액 (원화 또는 달러)
  - 손익 금액 (원화 또는 달러)
  - 손익률 (%)
  - 손익 상태 표시 (플러스/마이너스)
- 전체 포트폴리오 요약 (통합)
  - **모든 금액 원화로 통합 표시**
  - 국장 총 투자금액 (원화)
  - 미장 총 투자금액 (달러 → 원화 환산)
  - 총 투자금액 (원화 통합)
  - 총 평가금액 (원화 통합)
  - 총 손익 (원화 통합)
  - 수익률 (%)
- 데이터 상태
  - 마지막 업데이트 시간
  - 현재 환율 (1 USD = xxx KRW)
  - 새로고침 버튼

**현재가 자동 조회**
- **야후 파이낸스 (Yahoo Finance) 통합**
  - 국장/미장 모두 지원 (하나의 소스로 통합)
  - 방식: 웹 크롤링
  - 데이터: 15-20분 지연 (무료 서비스 한계)
  - 인증: 불필요
  - **국장 (한국 주식)**
    - API 형식: 종목코드.KS (KOSPI), 종목코드.KQ (KOSDAQ)
    - 화면 표시: 기업명 (종목코드)
    - 예시: 삼성전자 (005930), 카카오 (035720)
  - **미장 (미국 주식)**
    - API 형식: 티커 심볼
    - 화면 표시: 기업명 (티커)
    - 예시: Apple (AAPL), Tesla (TSLA), NVIDIA (NVDA)
- **환율 정보**
  - USD/KRW 환율 자동 조회 (야후 파이낸스 또는 환율 API)
  - 개별 종목: 원래 통화로 표시 (국장-원화, 미장-달러)
  - 통합 포트폴리오 요약: 모두 원화로 환산하여 표시
- **자동 새로고침**
  - 페이지 접속 시마다 최신 데이터 조회
  - 선택적 자동 새로고침 (1분, 5분, 10분 간격)
  - 수동 새로고침 버튼 제공
  - 장 운영 시간 표시 (현재 장 중인지 확인)

### 4.2 저장 방식

#### 클라우드 데이터베이스 (Supabase)
- 어디서나 접근 가능
- 자동 백업

- 로그인/회원가입

#### 저장되는 데이터 (Supabase DB)

**1. 가계부 데이터**
- 수입 내역 (항목명, 금액, 입금일, 입금 완료 여부)
- 고정 지출 내역 (항목명, 금액, 출금일, 출금 완료 여부)
- 변동 지출 내역 (항목명, 금액, 예정일, 출금 완료 여부)
- 월별 데이터 구분

**2. 부채 관리 데이터**
- 대출 내역 (날짜, 금액, 내용)
- 상환 내역 (날짜, 금액, 내용)

**3. 주식 보유 정보**
- 시장 구분 (KR/US)
- 기업명 (예: 삼성전자, Apple)
- 종목코드/티커 (예: 005930, AAPL)
- 보유 수량
- 평균 매수가
- 통화 (KRW/USD)
- 매수일 (선택)

**4. 사용자 정보**
- 사용자 ID (Supabase Auth)
- 설정 정보 (테마, 자동 새로고침 등)

---

#### 저장되지 않는 데이터 (실시간 조회)

**주식 현재가 정보**
- 현재가 → 야후 파이낸스에서 매번 조회
- 평가금액 → 현재가 × 보유 수량 (계산)
- 손익 금액 → (현재가 - 평균 매수가) × 수량 (계산)
- 손익률 → ((현재가 - 평균 매수가) / 평균 매수가) × 100 (계산)
- **개별 종목은 원래 통화로 표시**

**환율 정보**
- USD/KRW 환율 → 매번 조회
- **통합 포트폴리오 요약에만 환율 적용**
  - 국장 총액 (원화) + 미장 총액 (달러 → 원화 환산)
  - 전체 포트폴리오를 원화로 통합 표시

**이유:**
- 항상 최신 정보 제공
- 저장 공간 절약
- 불필요한 히스토리 데이터 없음
- 개별 종목은 원래 통화로 보는게 더 직관적

#### 예상 데이터베이스 테이블 구조

**transactions (가계부)**
```sql
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- type: VARCHAR (income/fixed/variable)
- name: VARCHAR (항목명)
- amount: DECIMAL (금액)
- date: DATE (날짜)
- is_completed: BOOLEAN (완료 여부)
- month: VARCHAR (월 구분, 예: 2025-01)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

**debts (부채 관리)**
```sql
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- type: VARCHAR (borrow/repay)
- amount: DECIMAL (금액)
- description: TEXT (내용)
- date: DATE (날짜)
- created_at: TIMESTAMP
```

**stocks (주식 보유)**
```sql
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- market: VARCHAR (KR/US)
- stock_name: VARCHAR (기업명)
- stock_code: VARCHAR (종목코드/티커)
- quantity: INTEGER (보유 수량)
- avg_price: DECIMAL (평균 매수가)
- currency: VARCHAR (KRW/USD)
- purchase_date: DATE (매수일, 선택)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

**settings (사용자 설정)**
```sql
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- theme: VARCHAR (light/dark)
- auto_refresh_interval: INTEGER (자동 새로고침 간격, 초)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

## 5. 개발 우선순위

### Phase 1 (MVP)
1. 가계부 기본 기능
   - 수입/지출 등록
   - 출금 체크
   - 월별 수지 계산
2. 부채 관리 기본 기능
   - 대출/상환 내역 등록
   - 잔액 계산
3. 주식 관리 기본 기능
   - 보유 주식 등록 (국장/미장 구분)
     - 국장 입력: 삼성전자 (005930), NAVER (035420)
     - 미장 입력: Apple (AAPL), Tesla (TSLA)
   - 현재가 자동 조회 (야후 파이낸스 웹 크롤링)
     - 국장: 종목코드.KS / .KQ 형식으로 API 조회
     - 미장: 티커 심볼로 API 조회
     - 접속 시마다 최신 데이터 조회
   - 손익 계산 및 표시
     - 개별 종목: 원래 통화로 표시 (국장-원화, 미장-달러)
     - 화면 표시: 기업명 (코드/티커) 형식
   - 환율 정보 조회
     - 통합 포트폴리오 요약에만 환율 적용 (전체를 원화로 통합)

### Phase 2
1. 통계 기능 추가
2. UI/UX 개선
3. 주식 자동 새로고침 기능

   - JSON 파일로 내보내기 (백업)

   - 데이터 통계 표시

### Phase 3
1. 주식 고급 기능
   - 실시간 데이터 연동 (증권사 API)
   - 주식 거래 내역 관리
   - 배당금 관리
   - 주식 알림 (목표가 도달 시)

#### 여러 PC에서 작업하는 경우

2. **작업 흐름**
   - PC A에서 작업 → `git push` (GitHub에 저장)
   - PC B로 이동 → `git pull` (최신 코드 가져오기) → 작업 → `git push`
   - PC A로 돌아옴 → `git pull` → 작업 계속

3. **주의사항**
   - 작업 시작 전: 항상 `git pull`
   - 작업 끝날 때: 항상 `git push`
   - 동시에 여러 PC에서 같은 파일 수정 금지

4. **환경변수 파일 (`.env`)**
   - GitHub에 올라가지 않음
   - 각 PC마다 수동으로 생성 필요
